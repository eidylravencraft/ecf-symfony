INSTALLATION :
- Installer la base de donnée fournie.
- clone le projet.
- composer install

Si pas d'installation de base de donnée :
- clone le projet
- symfony console make:migration
- symfony console d:s:u --force

Dans tous les cas :
- enjoy!

CE QUI MARCHE :
USER
- enregistrer un nouvel utilisateur
- s'identifier
- se déconnecter
- prendre un abonnement dans le pannel "Mon compte". Si l'abonnement expire, à la connexion de l'utilisateur, son status 'abonné' est enlevé et il n'a donc plus accès aux pages qui nécessite un abonnement. Il peut reprendre un abonnement.
- consulter l'historique des réservations de salle dans la partie "Mon compte".
SALLES
- Consulter la liste des salles (depuis la navbar)
- voir le planning des salles (refresh la page si le planning se s'affiche pas dans le détail)
- un clic sur une heure réserve à cet utilisateur le créneau. Un clic-glissé réserve plusieurs heures. Un clic sur un créneau déjà réservé pour éditer la durée ou le supprimer.

Nos migrations marchent, mais pour les salles que Lydie a créée et qui sont en dur dans la base de données (pas faites en fixtures) on fournit la base de données. Tu peux vérifier les migrations avant, mais il faudra malheureusement import la bdd pour avoir la liste des salles. Sinon tu peux créer des salles dans le backoffice.

Si tu ne fais pas d'import de bdd, après avoir crée l'user, il faut mettre le role à jour dans phpMyAdmin avec :
[
    "ROLE_USER",
    "ROLE_ADMIN"
]

Normalement ça devrait marcher.


Les routes sont bloquées depuis le security.yaml et certaines directement sur le controller.
